        HandshakeIdleTimeout: timeout / 2,

    This gives the QUIC handshake a total window of `2 * (timeout/2) = timeout`, matching the expected behavior and consistency with other probes.

    Additionally, wrap the `quic.DialAddr` call with a context deadline to ensure the QUIC probe cannot exceed the intended timeout even if quic-go's internal timeout calculation changes:

    dialCtx, dialCancel := context.WithTimeout(ctx, timeout)
    defer dialCancel()
    conn, err := quic.DialAddr(dialCtx, address, tlsConf, quicConf)

    This provides a hard deadline at the context level — the `dialCtx` will cancel after `timeout` regardless of quic-go's internal state machine. This is a defense-in-depth measure that also satisfies part of the context deadline propagation requirement (F-03).

    **Per D-06 and D-07:** Implement context deadline propagation in `scanTarget()`. Before each probe call, derive a child context with a bounded timeout from the remaining parent context deadline.

    **Step 1 — Add helper function for context derivation:**
    Add a helper at the bottom of scanner.go:
    ```go
    // probeContext derives a child context with a timeout for a single probe call.
    // share is the even division of remaining parent time across remaining probes.
    func probeContext(ctx context.Context, timeout time.Duration) (context.Context, context.CancelFunc) {
        deadline, ok := ctx.Deadline()
        if !ok {
            return context.WithTimeout(ctx, timeout)
        }
        remaining := time.Until(deadline)
        if remaining <= 0 {
            return context.WithTimeout(ctx, 1*time.Millisecond) // already expired, near-immediate cancel
        }
        // Use the minimum of the configured per-probe timeout and the remaining deadline.
        probeTimeout := timeout
        if remaining < probeTimeout {
            probeTimeout = remaining
        }
        return context.WithTimeout(ctx, probeTimeout)
    }
    ```

    **Step 2 — Wrap each probe call in scanTarget with derived context:**
    For each probe call, derive a child context and pass it instead of the raw `ctx`. The timeout value per probe is `options.Timeout` (the configured per-probe timeout). The derived context will use the minimum of `options.Timeout` and the remaining parent deadline.

    Modify the DNS probe calls (lines 85-90):
    ```go
    for _, resolver := range resolvers {
        if ctx.Err() != nil {

        probeCtx2, cancel2 := probeContext(ctx, options.Timeout)
        result.DNS = append(result.DNS, probeDNS(probeCtx2, resolver, target.Domain, mdns.TypeAAAA, options.Timeout))
        cancel2()
    }
    ```

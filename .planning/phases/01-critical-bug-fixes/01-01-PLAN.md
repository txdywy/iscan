    **Step 1 — Add Mismatch to TraceHop (model/model.go, per D-03):**
    Add `Mismatch bool \`json:"mismatch"\`` as a new field on TraceHop struct, after the Error field. Default zero-value false means "no mismatch detected" — matches existing behavior for Echo replies.

    **Step 2 — Replace PID-based ICMP ID (trace.go, per D-01):**
    - Remove `"os"` import.
    - Add `"crypto/rand"` and `"encoding/binary"` imports.
    - In `Probe()`, before the TTL loop, generate a random 16-bit ICMP Echo identifier:
      ```go
      var idBytes [2]byte
      if _, err := rand.Read(idBytes[:]); err != nil {
          observation.Error = "icmp id generation failed: " + err.Error()
          return observation
      }
      probeID := int(binary.BigEndian.Uint16(idBytes[:]))
      ```
    - Pass `probeID` as a new parameter to `probeHop()`.
    - Remove `myID := os.Getpid() & 0xffff` from `probeHop` — use the passed `probeID` instead.
    - Update `probeHop` signature to: `func probeHop(ctx context.Context, conn *icmp.PacketConn, ip net.IP, ttl int, timeout time.Duration, probeID int) (model.TraceHop, bool)`

    **Step 3 — Validate inner ICMP body in TimeExceeded (trace.go, per D-02):**
    In `probeHop`, in the `case *icmp.TimeExceeded:` branch (currently lines 124-133):
    - After parsing the TimeExceeded response, extract `body.Data` (the original packet that caused the timeout).
    - The inner packet is: [inner IP header (20+ bytes)] + [inner ICMP message header (8 bytes)].
    - Parse the inner IP header to get its length: `innerIHL := int(body.Data[0]&0x0f) * 4`
    - If `len(body.Data) < innerIHL + 8`, skip validation (inner packet truncated by router) — accept hop without Mismatch.
    - Otherwise, extract the inner ICMP header from `body.Data[innerIHL:]`.
    - The inner ICMP Echo header is 8 bytes: Type(1) + Code(1) + Checksum(2) + ID(2) + Seq(2).
    - Extract `innerID := int(binary.BigEndian.Uint16(body.Data[innerIHL+4 : innerIHL+6]))`
    - Extract `innerSeq := int(binary.BigEndian.Uint16(body.Data[innerIHL+6 : innerIHL+8]))`
    - Set `hop.Mismatch = (innerID != probeID || innerSeq != ttl)`.
    - Always accept the hop address (do NOT reject on mismatch) — return the hop with Mismatch flag set.
    - Set `hop.Address = peer.String()` and `hop.RTT = time.Since(sent)` as before.

    **Existing Echo validation (lines 119-123):** Keep the existing ID/Seq check for Echo replies but ALSO set `hop.Mismatch` when they mismatch (currently returns immediately without setting Mismatch — update to set `hop.Mismatch = true` and return with the hop address rather than rejecting outright).

    **Step 4 — Update Probe() call sites:**
    The call at trace.go line 61 `probeHop(ctx, conn, ip, ttl, timeout)` must be updated to `probeHop(ctx, conn, ip, ttl, timeout, probeID)`.

    **Current state analysis:** The current code at line 103 does `conn.SetDeadline(time.Now().Add(timeout))` which IS called per-hop iteration — each call computes a new deadline from `time.Now()`. The per-hop deadline is already independent. However, the per-hop timeout value equals the probe-level timeout (e.g., 5s), meaning a 30-hop trace could theoretically take up to 150s. The context cancellation check at the top of `probeHop` (lines 84-88) provides an escape hatch if the parent context expires.

    **Fix — Add per-hop timeout cap with explicit independence (P3):**
    - In `Probe()`, before the TTL loop, derive a per-hop timeout: `hopTimeout := min(timeout, 2*time.Second)`. This caps each hop at 2s regardless of the probe-level timeout, preventing cumulative timeout blowup.
    - Pass `hopTimeout` to each `probeHop` call instead of the full `timeout`.
    - In `probeHop`, keep the existing `conn.SetDeadline(time.Now().Add(hopTimeout))` pattern — it is already correct.
    - Add a comment on the `SetDeadline` line: `// Independent per-hop deadline; each iteration starts a fresh timer.`
    - After `conn.ReadFrom(reply)`, check for net.Error with Timeout() and return an explicit timeout error string for clarity.
    - Verify the context cancellation check at `probeHop` lines 84-88 remains intact — it provides the ultimate deadline enforcement from the parent context.

    **Consecutive timeout tracking:** The existing `consecutiveEmpty` logic at line 63-67 correctly handles back-to-back timeouts. No changes needed here — this mechanism works because each hop independently times out.

    **Latency timer start:** The `start := time.Now()` at line 17 is correct — the latency timer starts before DNS resolution. Per PITFALLS P3, DNS latency should NOT be included in trace latency, but per D-01 through D-11 there is no decision to change this. Leave as-is.

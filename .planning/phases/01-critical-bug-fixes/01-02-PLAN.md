<objective>
Replace the shared errgroup context cancellation with per-target independent context isolation. One target's probe failure must not cancel scanning of other targets. Collect target errors in the results slice instead of via errgroup.Wait().

Purpose: Fix cascading cancellation bug (PITFALLS P16) where errgroup.WithContext cancels the derived context on first non-nil error, causing all remaining target scans to be aborted. This produces incomplete scan reports where some targets are never scanned.
Output: Scanner that isolates per-target failures. All targets are scanned regardless of sibling failures. Failed targets include partial results + error.
</objective>

    **Step 1 — Add Error field to TargetResult (model/model.go, per D-05):**
    Add `Error string \`json:"error,omitempty"\`` as a new field on TargetResult struct. This field holds a per-target error message when a target scan fails (leaving partial results intact). Add it after the `Target` field.

    **Step 2 — Replace errgroup.WithContext with plain errgroup + per-target context.WithCancel (scanner.go, per D-04):**
    - Change `group, gCtx := errgroup.WithContext(ctx)` to `var group errgroup.Group` (no derived context — the group itself no longer carries cancellation).
    - In the goroutine closure, create a per-target context: `targetCtx, targetCancel := context.WithCancel(ctx)` and `defer targetCancel()`.
    - Pass `targetCtx` (NOT gCtx) to `scanTarget()`.
    - Remove the `select { case <-gCtx.Done(): ... }` gate since gCtx no longer exists. The per-target `targetCtx` will be checked inside `scanTarget` and `retryWithBackoff` via `ctx.Done()` checks.
    - Keep `group.SetLimit(options.Parallelism)` — concurrency limiting is still valuable and not the source of the bug.

    **Step 3 — Collect per-target errors in TargetResult.Error (scanner.go, per D-05):**
    - Modify the goroutine to capture scan errors in the result rather than returning them from the errgroup callback:
      ```go
      group.Go(func() error {
          targetCtx, targetCancel := context.WithCancel(ctx)
          defer targetCancel()
          result := scanTarget(targetCtx, target, resolvers, options)
          result.Findings = classifier.Classify(result)
          results[i] = result
          return nil // always return nil — errors are in result.Error
      })
      ```
    - The goroutine always returns `nil` from the errgroup callback. Per-target errors surface through `TargetResult.Error`.
    - The `group.Wait()` error check at line 65 now only catches fatal errors (panic recovery, resource exhaustion) — not per-target probe failures.
    - Keep the `err != context.Canceled` guard but narrow the warning message to indicate a scanner-level issue rather than per-target: `"scanner: " + err.Error()`.

    **Step 4 — Update error checking in scanTarget to populate result.Error:**
    At the end of `scanTarget`, if the context was cancelled or the target experienced a fatal error before completing all probes, set `result.Error` appropriately. The function already returns partial results naturally — the key change is that `result.Error` now carries the error string for downstream consumers.

    **Step 5 — Update the post-scan loop (lines 68-77):**
    The existing loop skips targets where `result.Target.Name == ""` (cancelled before scan). Since we no longer cancel sibling targets, all targets should have a name. Keep the guard for safety but it should rarely trigger. Update the warning collection to also surface `result.Error` fields:
    ```go
    for i, result := range results {
        if result.Target.Name == "" {
            continue
        }
        if result.Error != "" {
            report.Warnings = append(report.Warnings, targetList[i].Domain+": "+result.Error)
        }
        // ... existing trace permission check ...
        report.Findings = append(report.Findings, result.Findings...)
        report.Targets = append(report.Targets, result)
    }
    ```

    **Import changes:** Remove `"golang.org/x/sync/errgroup"` if errgroup is no longer needed — but keep it since we still use `errgroup.Group` for concurrency limiting. We just no longer use `errgroup.WithContext`.

    ```go
    func TestTargetFailureDoesNotCancelOthers(t *testing.T) {
        // Run a scan with a short timeout against the real builtin target set.
        // One target's slow response should not cancel other targets.
        report := scanner.Run(t.Context(), model.ScanOptions{
            Timeout: 500 * time.Millisecond,
            Retries: 1,
            Trace:   false,
            QUIC:    false,
        })
        // Verify multiple targets were attempted (not all cancelled after first failure).
        if len(report.Targets) < 2 {
            t.Fatalf("expected at least 2 target results, got %d — first failure may have cancelled others", len(report.Targets))
        }
        // Verify no target result has an empty name (cancelled-before-scan stub).
        for i, result := range report.Targets {
            if result.Target.Name == "" {
                t.Errorf("target %d has empty name — was cancelled before scan started", i)
            }
        }
        if report.Duration <= 0 {
            t.Fatal("expected positive duration")
        }
        t.Logf("scanned %d targets in %v", len(report.Targets), report.Duration)
    }
    ```

    **Guard against offline environments:** Add a `t.Skip` call if the test detects no network connectivity (check via `net.LookupHost("example.com")` at test start). This follows the existing pattern where scanner tests depend on live network.

    **Note on test reliability:** This is an integration-level smoke test, consistent with the existing `TestBuildScanReportSkipsCancelledTargets` pattern. It verifies the cancellation isolation behavior against real infrastructure. For CI environments without network, the test gracefully skips.

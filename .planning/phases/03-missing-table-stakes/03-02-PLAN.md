```go
// FileSource loads targets from a JSON file on disk.
type FileSource struct {
    Path string
}

func (f FileSource) Load() ([]model.Target, error) {
    data, err := os.ReadFile(f.Path)
    if err != nil {
        return nil, fmt.Errorf("reading target set %q: %w", f.Path, err)
    }
    var targets []model.Target
    if err := json.Unmarshal(data, &targets); err != nil {
        return nil, fmt.Errorf("parsing target set %q: %w", f.Path, err)
    }
    if len(targets) == 0 {
        return nil, fmt.Errorf("target set %q: file contains zero targets", f.Path)
    }
    for i := range targets {
        if err := targets[i].Validate(); err != nil {
            return nil, fmt.Errorf("target set %q: target %d (%s): %w", f.Path, i, targets[i].Name, err)
        }
    }
    return targets, nil
}
```

The FileSource validates every target via `model.Target.Validate()` and rejects the entire file on any validation failure (fail-fast, clear error message). Per D-04, the JSON format is `[{"name":"custom","domain":"example.org","scheme":"https","ports":[443]}]` — a raw array of `model.Target` structs using existing json tags.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI args -> FileSource.Load() | User-supplied file path crosses into file I/O |
| JSON file content -> json.Unmarshal | Untrusted JSON data parsed into model.Target structs |
| FileSource.Load() -> scanner.Run() | Loaded targets injected into scanner pipeline |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04 | Tampering | FileSource.Load() file read | mitigate | os.ReadFile reads the file once; no TOCTOU window for this diagnostic tool. File content is user-supplied — user controls risk. |
| T-03-05 | Spoofing | JSON array into model.Target | mitigate | json.Unmarshal deserializes into typed struct. Validate() checks Name, Domain, Scheme, Ports on every entry. Malformed JSON returns parse error with file path. |
| T-03-06 | InfoDisc | FileSource error messages | accept | Error messages include file path — diagnostic tool running locally, no network exposure of paths |
| T-03-07 | Denial | Large JSON file | accept | No size limit enforcement. User controls the file — local CLI tool, self-inflicted risk |
